2. Our Privacy Guarantees

These are not promises — they are enforced by the app's architecture. There is no code path that transmits your data anywhere.

No server. Ever.
LUNA has no backend server. No API endpoints. No account system. Nothing to breach.
100% offline
Works without internet. All predictions run locally in the Rust core. No cloud required.
AES-256-GCM encrypted
All data encrypted at rest. Key derived from your PIN via Argon2id. We cannot decrypt it even if we wanted to.
No account
No sign-up. No email. No username. No password stored on any server.
Zero telemetry
No analytics SDK. No crash reporter. No usage statistics sent anywhere. Ever.
No third-party SDKs
No Firebase. No Crashlytics. No Facebook SDK. No ad networks. Zero dependencies on external services.
Panic wipe
Three-tap emergency delete in Settings. Wipes all data instantly, including the encryption key. Unrecoverable.
Open source
Every line of code is on GitHub. Audit it yourself.

3. Your Data — Stays Local

LUNA does not collect your data. The word "collect" implies data leaving your device and reaching a third party. That never happens here — not by policy, but by architecture.

We have zero access to anything you enter into LUNA. We cannot read it, keep it, sell it, or share it. There is no server to receive it. There is no network request to transmit it. The app is technically incapable of sending your data anywhere.

# What happens when you log a cycle date in LUNA: # 1. You type it in the app # 2. The Rust core encrypts it with AES-256-GCM (key = your PIN via Argon2id) # 3. The ciphertext is written to SQLite on YOUR device # 4. Nothing else happens. No sync. No upload. No ping. # # Macaron Software sees: nothing. # Apple / Google see: nothing (unless you opt into their health platforms).

The data that you store on your device:

  •   Cycle dates, period flow intensity, predicted dates
  •   Daily symptoms, mood, energy, basal body temperature
  •   Notes you write
  •   Notification preferences

All of this lives in an AES-256-GCM encrypted SQLite database on your phone. No one — not us, not Apple, not Google, not any third party — has access to it. Deleting the app destroys the data permanently.

What we never access, store, or transmit:

  • ✗ Device identifiers, IDFA, advertising IDs, GAID
  • ✗ Location data of any kind
  • ✗ Contact information, email, phone number
  • ✗ Browsing history or cross-app activity
  • ✗ Crash reports or usage analytics sent externally
  • ✗ IP address — no server means no server logs
  • ✗ Any biometric data outside the device secure enclave
  • ✗ Anything at all, ever

4. How Data Is Stored

All data is stored locally on your device in an AES-256-GCM encrypted SQLite database (via SQLCipher), compressed with zstd. The encryption key is derived from your PIN using Argon2id (winner of the Password Hashing Competition).

// Rust core — vault.rs let key = argon2id_derive(pin, salt, memory_cost=65536, iterations=3); let db = sqlite_open_encrypted(path, key, cipher="aes-256-gcm");

Optional encrypted backup (iCloud / Google Drive) uses the same key — you must re-enter your PIN to restore. We never have access to your backup.

5. HealthKit & Health Connect

iOS — HealthKit: If you grant permission, LUNA can read/write menstrual cycle data to Apple Health. This stays on your device and is subject to Apple's HealthKit policies. Revoke any time: iOS Settings → Health → Data Access → LUNA.

Android — Health Connect: If you grant permission, LUNA can sync cycle data with Health Connect. Revoke any time: Android Settings → Health Connect → App permissions → LUNA.

Neither integration sends data to any LUNA server. There is no LUNA server.

We never share, sell, license, or transmit your personal health data to:

7. Your Rights (GDPR / CCPA / PIPEDA)

Since all data is local and in your control, you can exercise every privacy right directly in the app:

Access
Open the app — all data is right there
Delete
Settings → Wipe All Data (irreversible)
Export
Settings → Export Data (JSON / CSV)
Portability
Encrypted backup + CSV export
Rectification
Edit any entry directly in the app
Restriction
Disable any health integration in Settings

Under GDPR, you have the right to access, rectify, erase, restrict, and port your data. Since all data is local, you exercise these rights directly in the app. There is no EU data protection officer to contact because no personal data is ever transmitted.

Legal basis for processing: Consent (Art. 6(1)(a)) — you enter data voluntarily. Special category health data: Explicit consent (Art. 9(2)(a)).

California residents: LUNA does not sell personal information. We do not "share" data for cross-context behavioral advertising. LUNA stores no personal data on any server. The only data that exists is what you enter, stored encrypted on your own device. There is nothing for us to disclose because we have no access to it.

Canadian residents: LUNA stores no personal data on any server and transfers no data outside your device. All health information is stored locally, encrypted, under your sole control. There is no data processing by Macaron Software — we have no technical means to access your information. You may access, correct, or delete your data directly within the app at any time.

8. Children

LUNA is intended for users aged 17 and older and is rated accordingly on the App Store and Google Play. LUNA does not collect any data from any user, including minors. Since there is no data transmission, no server, and no account system, no parental consent mechanism is required or applicable.
